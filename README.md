# Media Viewer 360 — Databricks Lakehouse for Streaming Media
An end-to-end data platform demonstrating Bronze → Silver → Gold → ML → Serving on
Databricks Lakehouse. Built around a streaming-media use case: ingest viewer events
and content metadata from S3, build a Viewer 360 with identity resolution, predict
subscriber churn with MLflow, and ship behind GitHub Actions CI/CD.
Why this repo
Most "data engineering portfolio" projects stop at one notebook. This one shows the
production motion: ingestion through governance through ML through deployment, with
the engineering hygiene (tests, CI/CD, docs) you'd expect in a real customer engagement.

What this demonstrates
PySpark + SQL fluency — Python for ingestion and ML, SQL for transformations
Identity resolution — deterministic + fallback rules across event streams
MLflow lifecycle — experiment tracking → registry → batch scoring + monitoring
CI/CD for data — Databricks Asset Bundles deployed by GitHub Actions
Production hygiene — unit tests, data quality checks, runbook for day-2 ops

Notable design decisions
See [docs/architecture.md](docs/architecture.md) for full rationale. Highlights:
Auto Loader over batch read — handles late-arriving events and schema drift
Identity resolution as a SQL view, not a table — easy to evolve rules without backfill
Feature Store from gold.viewer_360 — keeps features governed and reusable
Batch model scoring, not real-time — matches the use case (daily churn dashboards),
saves on serving cost. Switch to Model Serving when latency requirements demand it.
